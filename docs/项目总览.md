## 一、这是什么游戏

**《碧落黄泉》**（曾用/别名《我要上天》）——原创**神话·上天入地**题材的**竖屏手机动作游戏**（横版清关 / 割草 Beat-em-up）。

**一句话剧情**：她无病无灾，一夜魂散。他不信命，提枪**下黄泉、上碧落**——翻遍地府的生死簿，杀穿天庭的南天门，只为向那随手一笔便定人生死的神，讨一个「凭什么」。

**三界结构**（每章一界）：

| 章节 | 界域 | 玩法 | 状态 |
|---|---|---|---|
| 第一章 | **人间** | 10 关横版清关，第 10 关许褚 Boss | ✅ 可玩 |
| 第二章 | **地府** | 投井下坠 → 落水潜水（方案A） | 🚧 开发中 |
| 第三章 | **天庭** | 未开始 | 🔒 待做 |

> ⚠️ **美术是占位**：现在的赵云 / 许褚 / 曹字旗是**三国素材占位**，最终要按剧本换成神话角色。三国元素已废弃，别再往三国方向做内容。
> ⚠️ `docs/剧本.md`（抗日游击题材）是**早期废弃草案**，**采用的是** [`docs/剧本-碧落黄泉.md`](剧本-碧落黄泉.md)。

## 二、做到哪了（现状）

- **第一章（人间·战斗）**：完整可玩。10 个 zone，4 兵种（步兵/盾兵/精英/弓手）+ 许褚 Boss，三段连招（下劈/上挑/跳劈）+ 剑气 + 跳跃，关间三选一升级，昼夜/天气/三界染色系统。**手机稳 60 FPS**（优化实录见专题文档）。
  - ⚠️ 半成品：金币掉落了但**无用途**；XP+升级卡逻辑写好但**部分未接线**；弓手数值配好但**未入正式刷怪池**。详见 [`战斗玩法设计方案.md`](战斗玩法设计方案.md)。
- **第二章（地府·投井）**：方案A（落水潜水）已接进游戏、可跑通。井口正中坠落 → 砸破水面 → 水下浮力潜行；左侧石台可砸石开侧道。角色套件已抽成 `HeroRig` 与第一章对齐（走/待机/空中/横斩/跳劈 + 剑气/连招/刀气）。
  - 🔒 未做：井底 Boss、鬼门关过场、关卡目标与结算。
- **第三章（天庭）**：仅有剧本与界域染色预留，无实装。

**当前默认进入哪一章**由 [`GameRoot.ts`](../assets/scripts/GameRoot.ts) 的 `START_CHAPTER2` 开关决定（见「如何运行」）。

## 三、技术架构

- **引擎**：Cocos Creator 3.x，**TypeScript**，`_decorator` / `@ccclass` 组件体系。
- **分辨率**：固定设计分辨率 **720×1280 竖屏**（`Constants.ts` 的 `DESIGN_W/DESIGN_H`），大屏留黑边不变形。
- **纯代码驱动**：**没有场景文件**。挂一个 `GameRoot` 组件在 Canvas 下的空节点上，运行时**用代码创建所有节点/组件**。渲染顺序 = 创建顺序。
- **绘制**：大量用 `Graphics`（矢量描边填充）+ `Sprite`（序列帧切图）。⚠️ **Cocos `Graphics` 不支持渐变**——渐变要靠「分带填充」模拟（见第二章水体、井口透明淡出用烘焙 PNG）。
- **性能**：手机（微信小游戏）从 17 FPS 优化到稳 60。**新特效/新章节都要照** [`性能优化套路.md`](性能优化套路.md) **做**（精灵池、离屏烘焙、整数贴像素、避免每帧分配）。
- **分包与三界**：[`AssetHub.ts`](../assets/scripts/AssetHub.ts) 统一 `loadSF(name, cb)` 加载，查找顺序「当前界 Bundle → resources 兜底」。[`GameState.ts`](../assets/scripts/GameState.ts) 是全局状态单例（昼夜曲线等）。三界靠**一套底图染色**变出人间/地府/天庭氛围（`BattleScene` 的 `REALMS`），省 2/3 背景美术。

## 七、如何运行 / 切换章节

1. 用 **Cocos Creator 3.x** 打开本项目。
2. 运行预览（浏览器/模拟器）。开机 = 标题 → 点击进入。
3. **切章节**：改 [`GameRoot.ts`](../assets/scripts/GameRoot.ts)：
   - `const START_CHAPTER2 = true;` → 直接进**第二章·投井**。
   - 改成 `false` → 走**第一章**（标题→战场）。
   - 主城场景那一整段 `make(...)` 已注释；需要时解注即可恢复。
4. **加新图**：把 png/jpg 放进 `assets/resources/`，**切回编辑器窗口**让它自动导入（生成 `.meta`）后才能被 `loadSF` 加载。

## 八、接手须知（坑 & 约定）

- **提交约定（重要）**：用户说「**提交**」= **commit 并 push 到远端**,一条龙,直接在 `main` 上：
  ```bash
  git add -A && git commit -m "..." && git push
  ```
  ⚠️ **只 `commit` 不 `push` = 没完成**。用户默认「提交」就包含推远端;漏 push 会让改动卡在本机、另一台拉不到。
  若 push 被 `rejected`(历史分叉),先 `git pull --rebase` 再 `git push`。
- **美术全是占位**：任何三国角色都要按剧本换神话角色，别在占位美术上堆死逻辑。
- **美术目标风格 = Q版可爱**（2026-07-17 定，第一章也换）：所有 AI 生成图必须带风格锚、挂同一参考图。详见 [`美术风格锚-Q版.md`](美术风格锚-Q版.md)。
- **两套坐标系**：
  - 第一章 `BattleScene`：横版，世界 x 随镜头卷动。
  - 第二章 `Chapter2Well`：竖版下坠，**世界 y 向下为正**，用 `SX/SY` 把 demo 坐标（460×760）缩放贴到 Cocos 居中坐标。角色/特效是**绝对像素尺寸**（×1.8），场景按 `SCALE` 缩放——两者独立。
- **`Graphics` 无渐变**：渐变一律分带模拟；需要真渐变（如透明淡出）就用 Python 把 alpha **烘进 PNG**。
- **异步加载顺序会咬人**：小图可能比大图先加载，后加载的会盖在先加载的上面。第二章为此把井口/井台放独立 `decorRoot` 容器固定层级——加新贴图注意层级。
- **性能红线**：见 [`性能优化套路.md`](性能优化套路.md)。核心：精灵池复用、离屏烘焙静态像素、整数对齐防抖、`update` 里零分配。

## 九、下一步路线（建议）

- **第一章增时长+增趣味**（用户最新诉求）：现在 ≈62 杂兵 5–8 分钟通关偏短。方向：①敌人种类翻倍 + 精英变小 Boss（改 `ZONE_PLAN` 数据表最省事）②Roguelite 化（分支地图 + 升级池扩到 20–30 张 + 局外成长）③玩家加动词（冲刺/格挡/处决，复用 `HeroRig`）。头脑风暴池见 [`战斗玩法设计方案.md`](战斗玩法设计方案.md)。
- **第二章**：井底 Boss、鬼门关过场、关卡目标与结算。
- **接线半成品**：金币用途、升级卡完全接通、弓手入池。
- **第三章·天庭**：`ZONE_REALM` 把对应关卡改 `'heaven'` 即可套用染色。
